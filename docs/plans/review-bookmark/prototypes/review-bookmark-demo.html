내 답변

프로세스는 운영체제에서 실행 중인 프로그램의 인스턴스이고, 스레드는 프로세스 내에서 실행되는 작업 단위입니다. 프로세스는 독립적인 메모리 공간을 가지고 있고 스레드는 같은 프로세스 내에서 메모리를 공유합니다.

모범 답변

프로세스는 OS가 할당하는 독립 실행 단위로, 자체 가상 주소 공간(Code/Data/Heap/Stack)을 보유합니다. 스레드는 프로세스 내부의 실행 흐름으로, 코드·데이터·힙은 공유하되 스택만 독립적입니다. 컨텍스트 스위칭 비용은 스레드가 낮고, 프로세스 간 통신은 IPC가 필요합니다.

AI 코칭

스레드 간 메모리 공유의 장단점(동기화 비용, 경쟁 상태)을 함께 설명하면 더 깊이 있는 답변이 됩니다.

내 답변

TCP는 연결 지향적 프로토콜이고 UDP는 비연결 프로토콜입니다. TCP는 신뢰성이 높고 순서를 보장하며, UDP는 빠르지만 데이터 손실이 발생할 수 있습니다.

모범 답변

TCP는 3-way handshake로 연결을 수립한 뒤 흐름/혼잡 제어를 수행하는 연결 지향 프로토콜입니다. UDP는 비연결로 핸드셰이크 없이 바로 데이터를 전송합니다. HTTP/HTTPS·이메일·파일 전송은 TCP, 게임·실시간 스트리밍·DNS는 UDP를 사용합니다.
