내 답변

HashMap은 순서를 보장하지 않고, LinkedHashMap은 삽입 순서를 유지합니다. 둘 다 key-value 쌍을 저장하며 null 키를 허용합니다.

모범 답변

HashMap은 해시 테이블 기반으로 순서를 보장하지 않으며 O(1) 접근이 가능합니다. LinkedHashMap은 HashMap을 확장하여 이중 연결 리스트로 삽입 순서 혹은 접근 순서를 유지합니다. 순서가 필요한 LRU 캐시 구현에는 LinkedHashMap이 적합합니다.

AI 코칭

접근 순서 유지 기능과 LRU 캐시 활용 사례까지 언급하면 더욱 완성도 높은 답변이 됩니다.

내 답변

프로세스는 독립된 메모리 공간을 가지고, 스레드는 같은 프로세스 내 메모리를 공유합니다. synchronized 키워드나 Lock을 사용해 동기화합니다.

모범 답변

프로세스는 OS 자원 할당 단위로 독립 메모리를 가집니다. 스레드는 프로세스 내 실행 단위로 힙·코드 영역을 공유합니다. 동기화는 synchronized, ReentrantLock, volatile, Atomic 클래스 등을 상황에 맞게 사용하며, 교착 상태(deadlock) 방지를 위해 락 순서를 일관되게 유지해야 합니다.

AI 코칭

deadlock 조건과 방지 전략, volatile vs synchronized 차이까지 확장하면 심화 답변이 됩니다.

언어·프레임워크

3

내 답변

Virtual DOM은 실제 DOM 변경을 최소화해서 빠릅니다. memo와 useCallback으로 리렌더링을 방지할 수 있습니다.

모범 답변

Virtual DOM은 메모리 내 가상 트리를 유지하며 diffing 알고리즘으로 변경점만 실제 DOM에 반영해 배치 업데이트 비용을 낮춥니다. 리렌더링 최적화는 React.memo(props 얕은 비교), useMemo(계산 메모이제이션), useCallback(함수 참조 유지), key 속성 최적화로 이루어집니다.

AI 코칭

diffing 알고리즘의 O(n) 휴리스틱, Reconciliation 과정, 그리고 key의 역할을 함께 설명하면 심층적인 이해를 보여줄 수 있습니다.

내 답변

Spring Bean은 컨테이너가 시작될 때 생성되고, 의존성 주입이 완료된 뒤 @PostConstruct가 실행됩니다. 컨테이너 종료 시 @PreDestroy가 호출됩니다.

모범 답변

Spring Bean의 생명주기: 인스턴스화 → 의존성 주입 → BeanNameAware·BeanFactoryAware 등 Aware 콜백 → BeanPostProcessor#postProcessBeforeInitialization → @PostConstruct / InitializingBean#afterPropertiesSet → BeanPostProcessor#postProcessAfterInitialization → 사용 → @PreDestroy / DisposableBean#destroy 순입니다.

AI 코칭

BeanPostProcessor의 역할과 프록시 생성 시점까지 언급하면 AOP 동작 원리와 연결되어 좋은 인상을 줄 수 있습니다.

내 답변

브라우저가 HTML 파싱 → DOM 생성 → CSS 파싱 → CSSOM → Render Tree → Layout → Paint 순으로 렌더링합니다. defer 속성과 이미지 lazy loading으로 개선할 수 있습니다.

모범 답변

모범 답변이 제공되지 않은 질문입니다.

인프라·클라우드

1

내 답변

Docker는 OS 커널을 공유하는 컨테이너 방식이고, VM은 하이퍼바이저 위에 게스트 OS를 올립니다. Docker가 가볍고 빠르며 이식성이 좋습니다.

모범 답변

VM은 하이퍼바이저(Type 1/2)가 하드웨어를 추상화해 각 VM이 전체 OS를 포함합니다. Docker는 Host OS 커널을 공유하므로 부팅 시간(ms 단위), 메모리 오버헤드가 현저히 낮습니다. 이식성(이미지 레이어), 환경 일관성, CI/CD 통합이 주요 이점이며, 보안 격리 측면에서는 VM이 여전히 강점을 가집니다.

AI 코칭

네임스페이스/cgroup 기반 격리 원리와 보안 격리의 한계(container escape)를 언급하면 깊이를 더할 수 있습니다.

행동·경험

1

내 답변

기술 스택 선택 문제로 의견이 갈렸을 때 각자 장단점을 문서로 정리하고 팀 회의에서 논의해 합의를 이끌어냈습니다.

모범 답변

STAR 구조로 답변하세요. Situation(상황 배경), Task(본인 역할), Action(구체적 행동 — 데이터 기반 논의 제안, 중립적 퍼실리테이션 등), Result(결과와 배운 점)를 명확히 구분하면 면접관이 역량을 평가하기 쉽습니다.

AI 코칭

구체적인 수치(일정 단축, 성능 수치 등)와 함께 본인이 취한 행동의 근거를 명확히 설명하면 설득력이 높아집니다.
