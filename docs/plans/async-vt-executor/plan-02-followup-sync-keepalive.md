# Plan 02: 후속질문 생성 — 동기 유지 (변경 없음)

> 상태: Draft
> 작성일: 2026-03-27

## Why

`generateFollowUp`은 면접 중 다음 질문을 기다리는 동기 API다. 외부 API 호출(~3초)은 FE가 응답을 대기하는 시간이므로, 비동기로 전환할 이유가 없다. 현재 트래픽(~8 req/s)에서 PT 200 스레드로 충분히 처리 가능하다.

## 변경 사항

**없음.** 현재 코드 그대로 유지한다.

### 현재 흐름 (유지)

```
[FE] POST /follow-up → [PT] generateFollowUp() → STT + AI API + DB → 200 응답
                         PT 3초 점유 (현재 트래픽에서 문제 없음)
```

### 유지 근거

| 근거 | 설명 |
|------|------|
| FE 응답 대기 필수 | 면접 중 다음 질문을 즉시 보여줘야 함 |
| 3초는 허용 범위 | PT 200 스레드 기준 66 req/s 처리 가능, 현재 ~8 req/s |
| TX 분리 이미 완료 | DB 커넥션 점유 3초→40ms, 커넥션 풀 고갈 없음 |
| 코드 복잡도 | 동기가 가장 단순하고 예외 처리가 명확함 |

### 향후 전환 가능성

트래픽이 66 req/s에 근접하면 `CompletableFuture<ResponseEntity<>>` 반환으로 전환 가능. FE 변경 없이 PT만 해제하는 패턴이다. 그때 plan 추가하면 된다.
