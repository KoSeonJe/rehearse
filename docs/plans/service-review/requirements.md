# 서비스 리뷰 — 요구사항 정의

> 상태: Draft
> 작성일: 2026-03-20

## Why

실사용 리뷰(노션 "전체 서비스 로직" 문서 12개 댓글)에서 품질/UX 이슈가 도출되었다.
사용자가 면접 완료 후 피드백·리포트를 확인하는 핵심 경로에서 무한로딩, 실패 미안내, 스킵 질문 노출 등의 문제가 발생하며 서비스 신뢰도를 떨어뜨린다.

### Why? — 어떤 문제를 해결하는가?

- 리포트 생성 경로가 이중(Lambda 직접 호출 + FE 요청)이라 race condition과 무한로딩 발생
- 스킵한 질문이 피드백 페이지에 노출되어 사용자 혼란
- 분석 실패/타임아웃 시 아무 안내 없이 대기 상태 지속
- 리포트 페이지에서 500 에러 시 무한 폴링
- 노션/로컬 문서에 삭제된 기능(VAD) 참조, 변경된 아키텍처 미반영

### Goal — 구체적인 결과물은?

- 리포트 생성 단일 경로 확보 (FE 요청만)
- 실패·에러 상황에서 즉각적인 사용자 안내
- 스킵 질문 완전 숨김
- 문서와 코드 간 불일치 제거

### Evidence — 근거는?

- 노션 댓글 12개 (실사용 리뷰에서 직접 도출)
- 댓글 9: 리포트 무한로딩 재현 확인
- 댓글 10: Lambda→BE 리포트 직접 호출로 인한 race condition 확인
- 댓글 7: 분석 실패 시 UI 무반응 확인

### Trade-offs — 포기하는 것은?

- Lambda에서 리포트 자동 생성 제거 → 사용자가 리포트 페이지를 방문해야 생성 시작 (지연 허용)
- 문서 업데이트를 별도 태스크로 분리 → 구현과 문서가 일시적으로 불일치할 수 있음

## Scope

- **In**: 리포트 경로 단일화, 스킵 숨김, 실패 UI, 리포트 에러 처리, 문서 동기화
- **Out**: AI 질문 품질 개선 (Sprint 1), 새로운 기능 추가, 인프라 변경
