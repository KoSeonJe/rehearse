## 1. 핵심 요약

### 프로젝트 개요

리허설(Rehearse)은 한국 개발자 취업시장을 타겟으로 하는 AI 기반 모의면접 플랫폼이다. 기존 서비스가 면접 종료 후 텍스트 점수만 제공하는 것과 달리, 리허설은 녹화된 면접 영상을 타임스탬프 단위로 분석하여 언어적/비언어적 피드백을 동시에 제공한다.

사용자는 녹화본을 재생하며 '이 시점에서 시선이 불안정했습니다', '여기서 답변 구조가 부족합니다' 같은 구체적 피드백을 해당 장면과 함께 확인할 수 있다.

### 핵심 차별점

| 구분 | 기존 서비스 | 리허설(Rehearse) |
|------|------------|---------|
| 피드백 방식 | 면접 종료 후 텍스트 점수표 | **타임스탬프 기반 영상+피드백 동기화** |
| 비언어적 분석 | 없음 또는 추상적 점수 | **실시간 시선/표정/자세 추적 + 이벤트 마커** |
| 개발자 특화 | 범용 면접 (BQ 중심) | **CS 기초 + 시스템 설계 + 코딩 면접** |
| 후속 질문 | 고정 질문 목록 | **AI 기반 맥락적 후속 질문** |
| 비용 | 서버 AI 추론 비용 발생 | **MediaPipe 비언어분석 $0 (브라우저 실행)** |

## 3. 제품 정의

### 서비스 컨셉

리허설은 **"면접 영상을 보면서 배우는 면접"** 이라는 컨셉으로, 기존의 '면접 보고 → 텍스트 점수 확인' 패턴을 '면접 보고 → 타임스탬프별 영상+피드백 동기화 확인'으로 전환한다.

### 사용자 여정 (핵심 플로우)

1. **면접 설정**: 이력서 업로드 또는 직무/레벨 선택 → 면접 유형 선택 (CS기초/시스템설계/BQ)
2. **AI 면접 진행**: AI 면접관이 질문 → 사용자 음성+영상 응답 → AI 후속질문 (3~5회 반복)
3. **실시간 비언어 추적**: MediaPipe가 면접 중 시선/표정/자세를 연속 분석하여 타임스탬프 이벤트 기록
4. **면접 종료 후 AI 분석**: 답변 내용 + 비언어 이벤트 통합 분석 → 타임스탬프별 피드백 생성
5. **피드백 리뷰**: 비디오 플레이어에서 타임스탬프 클릭 → 해당 장면 이동 + 언어/비언어 피드백 동시 표시

## 4. MVP 기능 정의

### Feature 1: AI 면접관 시스템

AI가 이력서와 직무정보를 기반으로 면접 질문을 생성하고, 사용자 답변을 실시간 분석하여 맥락적 후속 질문을 생성한다.

#### 면접 유형

| 면접 유형 | 질문 범위 | 평가 기준 |
|----------|----------|----------|
| CS 기초 | 자료구조, 알고리즘, OS, 네트워크, DB | 개념 정확성, 설명 구조화, 실무 연결력 |
| 시스템 설계 | 스케일링, 아키텍처 패턴, 트레이드오프 | 요구사항 분석력, 설계 근거, 확장성 고려 |
| Behavioral | STAR 기반 경험 질문, 협업/갈등/리더십 | 구체성, 역할 명확성, 성찰적 학습 |

#### 후속 질문 로직

AI 면접관은 사용자 답변을 STT로 실시간 변환한 후, 다음 기준으로 후속 질문을 결정한다:

- **답변 깊이 부족**: 핵심 개념에 대한 원리 설명 요청 ("그 알고리즘의 시간 복잡도가 왜 그렇게 되는지 설명해주세요")
- **답변 오류**: 잘못된 부분을 구체적으로 지적하며 재설명 요청
- **답변 우수**: 심화 질문으로 엘리베이션 ("트래픽이 100배 늘어나면 어떻게 대응하시겠습니까?")
- **비언어 이벤트 연계**: 장기간 시선 회피 감지 시 "다른 방식으로 설명해보시겠어요?"

### Feature 2: 실시간 비언어 분석 시스템

MediaPipe를 활용하여 브라우저에서 직접 실행되며, 서버 비용이 전혀 발생하지 않는다.

#### 분석 항목 및 이벤트 정의

| 분석 영역 | 추적 항목 | 이벤트 트리거 조건 | 피드백 예시 |
|----------|----------|------------------|-----------|
| 시선 | Face Mesh 468pt 동공좌표 | 카메라 중앙에서 3초+ 이탈 | "시선이 불안정합니다" |
| 표정 | Face Mesh 감정 추정 | 장기간 무표정/골혀 감지 | "표정이 경직되어 있습니다" |
| 자세 | Pose 33pt 랜드마크 | 어깨 기울임/머리 손 터치 | "자세가 기울어져 있습니다" |
| 음성 | Web Audio 음량/속도 | 3초+ 침묵/빠른 속도/얼- 반복 | "답변 속도가 빠릅니다" |

#### MediaPipe 기술 상세

- **Face Mesh**: 468개 얼굴 랜드마크를 30fps로 추적. 시선 방향, 표정 변화, 입 움직임 감지
- **Pose Landmark**: 33개 신체 포인트 추적. 어깨 기울기, 상체 자세, 손 위치 분석
- **실행 환경**: WebGL/WebAssembly 기반 브라우저 실행. 사용자 디바이스의 CPU/GPU 사용 → 서버 추론 비용 $0
- **처리 방식**: 매 프레임마다 분석 → 임계값 초과 시 이벤트 배열에 `{timestamp, type, severity, data}` 추가

### Feature 3: 타임스탬프 기반 피드백 리뷰

> **리허설의 핵심 차별점.** 면접 녹화본을 재생하며 언어적/비언어적 피드백을 타임스탬프에 동기화하여 보여준다.

#### UI 구성

- **좌측 60%**: 비디오 플레이어 — 면접 녹화본 재생, 타임라인에 이벤트 마커 표시
- **우측 40%**: 피드백 패널 — 현재 재생 지점의 피드백 표시
- **하단**: 타임라인 내비게이션 — 언어적 이벤트(파란색 마커) + 비언어적 이벤트(주황색 마커) 구분 표시

#### 이벤트 마커 유형

| 색상 | 유형 | 예시 | 심각도 | 클릭 시 |
|------|------|------|--------|--------|
| 🔵 파란 | 언어적 피드백 | 답변 구조 부족, 핵심 누락 | 높음/중간/낮음 | 해당시점 이동 |
| 🟠 주황 | 비언어적 피드백 | 시선 이탈, 표정 경직, 자세 불안정 | 높음/중간/낮음 | 해당시점 이동 |
| 🟢 초록 | 우수 구간 | 명확한 설명, 좋은 자세 유지 | - | 해당시점 이동 |

#### 타임라인 동기화 로직

비디오 플레이어의 `currentTime`과 이벤트 배열의 `timestamp`를 매칭하여 재생 위치에 해당하는 피드백을 실시간으로 표시한다.

- 이벤트 마커 클릭 → 해당 시점으로 비디오 점프 + 우측 패널에 피드백 표시
- 비디오 재생 중 → 현재 시점의 피드백이 우측 패널에 자동 표시
- 양방향 동기화: 비디오 ↔ 타임라인 ↔ 피드백 패널

## 5. MVP 스코프 정의

### MVP DO (반드시 포함)

| # | 기능 | 세부 사항 |
|---|------|----------|
| 1 | AI 면접관 | 이력서 기반 + CS/시스템설계/BQ 질문 생성, 후속 질문 |
| 2 | 영상 녹화 | MediaRecorder API (WebM), 로컬 저장 |
| 3 | 비언어 분석 | MediaPipe Face Mesh + Pose Landmark (브라우저) |
| 4 | 음성 분석 | Web Audio API 음량/속도/침묵 감지 |
| 5 | 타임스탬프 피드백 | 비디오+피드백 동기화 UI, 이벤트 마커 타임라인 |
| 6 | STT 변환 | Web Speech API / Whisper API 폴백 |
| 7 | AI 피드백 생성 | Claude API 답변+비언어 통합 분석 |
| 8 | 종합 리포트 | 면접별 요약 점수 + 개선포인트 + 강점 |

### MVP DON'T (후속 버전 연기)

- 기업별 질문 DB (RAG 기반)
- 사용자 면접이력 누적 대시보드
- 멀티유저 피어리뷰
- 모바일 앱 (웹 퍼스트)
- 결제 시스템 (베타 무료)
- 코딩 테스트 실행 환경 (IDE 통합)

```

                    │

                                          │

                                    ──→ 후속 질문 생성

[면접 종료 후]

                    │
                    ▼
              Claude API ──→ 타임스탬프별 피드백 JSON + 종합 요약 생성
                    │
                    ▼
              피드백 리뷰 UI에 렌더링
```

### 핵심 데이터 구조

```typescript
// 비언어 이벤트 (브라우저에서 MediaPipe로 수집)
interface NonVerbalEvent {
  timestamp: number;       // 비디오 시간 (초)
  type: 'gaze' | 'expression' | 'posture' | 'voice';
  severity: 'high' | 'medium' | 'low';
  data: {
    description: string;   // "시선이 좌측으로 이탈"
    value?: number;        // 정량값 (각도, 데시벨 등)
    duration?: number;     // 지속시간 (초)
  };
}

// 타임스탬프 피드백 (AI 생성 — 언어적/비언어적/콘텐츠 통합)
interface TimestampFeedback {
  id: number;
  interviewId: number;
  timestampSeconds: number;   // 비디오 시간 (초)
  category: 'VERBAL' | 'NON_VERBAL' | 'CONTENT';
  severity: 'HIGH' | 'MEDIUM' | 'LOW';
  content: string;            // 피드백 내용
  suggestion: string;         // 개선 제안
}

// 면접 세션 (Backend Entity 기준)
interface InterviewSession {
  id: number;
  position: string;           // 직무 (BACKEND, FRONTEND 등)
  level: string;              // 경력 수준
  interviewTypes: string[];   // 면접 유형 (CS, SYSTEM_DESIGN, BEHAVIORAL)
  durationMinutes: number;    // 면접 시간 (분)
  resumeText: string;         // 이력서 텍스트
  status: 'CREATED' | 'IN_PROGRESS' | 'COMPLETED';
  questions: Question[];
  createdAt: Date;
}
```

> **참고**: 영상은 현재 MVP에서 브라우저 로컬(Blob URL)에만 저장되며, 서버 업로드는 미구현 상태이다.

## 7. AI 프롬프트 아키텍처

> **RAG 없이 Claude의 기존 지식 + 잘 설계된 프롬프트로 MVP를 구현한다.**
> RAG는 기업별 질문DB나 사용자 이력 기반 피드백이 필요한 후속 버전부터 도입.

### 1단계: 질문 생성 프롬프트

- **입력**: 이력서 정보 + 면접 유형 + 난이도 설정
- **출력**: 면접 질문 5개 + 각 질문별 평가 루브릭
- **전략**: System prompt에 면접관 페르소나 설정 + 평가 기준 명시. Claude가 이미 알고 있는 CS 개념들을 활용하여 면접 질문 생성

### 2단계: 후속 질문 프롬프트

- **입력**: 이전 질문 + 사용자 답변 텍스트 + 평가 루브릭 + 비언어 이벤트(선택적)
- **출력**: 후속 질문 1개 + 이유
- **전략**: 답변 품질에 따라 분기 (심화/재질문/칭찬 후 다음 주제). 비언어 이벤트 연계하여 자연스러운 대화 유도

### 3단계: 통합 피드백 프롬프트

- **입력**: 전체 면접 대화 로그 + 전체 비언어 이벤트 배열 + 평가 루브릭
- **출력**: 타임스탬프별 피드백 배열 (JSON) + 종합 요약
- **전략**: 각 답변 구간에 대해 언어적 평가 + 해당 구간의 비언어 이벤트를 함께 분석하여 통합 피드백 생성
