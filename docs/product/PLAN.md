## 1. 핵심 요약

### 프로젝트 개요

리허설(Rehearse)은 한국 개발자 취업시장을 타겟으로 하는 AI 기반 모의면접 플랫폼이다. 기존 서비스가 면접 종료 후 텍스트 점수만 제공하는 것과 달리, 리허설은 녹화된 면접 영상을 타임스탬프 단위로 분석하여 언어적/비언어적 피드백을 동시에 제공한다.

사용자는 녹화본을 재생하며 '이 시점에서 시선이 불안정했습니다', '여기서 답변 구조가 부족합니다' 같은 구체적 피드백을 해당 장면과 함께 확인할 수 있다.

### 핵심 차별점

| 구분 | 기존 서비스 | 리허설(Rehearse) |
|------|------------|---------|
| 피드백 방식 | 면접 종료 후 텍스트 점수표 | **타임스탬프 기반 영상+피드백 동기화** |
| 비언어적 분석 | 없음 또는 추상적 점수 | **GPT-4o Vision 기반 시선/표정/자세 분석 + 타임스탬프 마커** |
| 개발자 특화 | 범용 면접 (BQ 중심) | **CS 기초 + 시스템 설계 + 코딩 면접** |
| 후속 질문 | 고정 질문 목록 | **AI 기반 맥락적 후속 질문** |
| 비용 | 서버 AI 추론 비용 발생 | **질문세트 단위 분석으로 효율적 비용 관리** |

## 3. 제품 정의

### 서비스 컨셉

리허설은 **"면접 영상을 보면서 배우는 면접"** 이라는 컨셉으로, 기존의 '면접 보고 → 텍스트 점수 확인' 패턴을 '면접 보고 → 타임스탬프별 영상+피드백 동기화 확인'으로 전환한다.

### 사용자 여정 (핵심 플로우)

1. **면접 설정**: 이력서 업로드 또는 직무/레벨 선택 → 면접 유형 선택 (CS기초/시스템설계/BQ)
2. **AI 면접 진행**: AI 면접관이 질문 → 사용자 음성+영상 응답 → AI 후속질문 (3~5회 반복)
3. **질문세트별 녹화 + S3 업로드**: 질문 세트 완료 시 녹화 영상을 S3에 업로드, 답변 타임스탬프 저장
4. **서버사이드 AI 분석**: S3 업로드 → EventBridge → Lambda 자동 트리거 → OpenAI Whisper STT + GPT-4o Vision 비언어 분석 + GPT-4o LLM 언어 분석 → 타임스탬프별 피드백 생성
5. **피드백 리뷰**: 비디오 플레이어에서 타임스탬프 클릭 → 해당 장면 이동 + 언어/비언어 피드백 동시 표시

## 4. MVP 기능 정의

### Feature 1: AI 면접관 시스템

AI가 이력서와 직무정보를 기반으로 면접 질문을 생성하고, 사용자 답변을 실시간 분석하여 맥락적 후속 질문을 생성한다.

#### 면접 유형

| 면접 유형 | 질문 범위 | 평가 기준 |
|----------|----------|----------|
| CS 기초 | 자료구조, 알고리즘, OS, 네트워크, DB | 개념 정확성, 설명 구조화, 실무 연결력 |
| 시스템 설계 | 스케일링, 아키텍처 패턴, 트레이드오프 | 요구사항 분석력, 설계 근거, 확장성 고려 |
| Behavioral | STAR 기반 경험 질문, 협업/갈등/리더십 | 구체성, 역할 명확성, 성찰적 학습 |

#### 후속 질문 로직

AI 면접관은 사용자 답변을 STT로 실시간 변환한 후, 다음 기준으로 후속 질문을 결정한다:

- **답변 깊이 부족**: 핵심 개념에 대한 원리 설명 요청 ("그 알고리즘의 시간 복잡도가 왜 그렇게 되는지 설명해주세요")
- **답변 오류**: 잘못된 부분을 구체적으로 지적하며 재설명 요청
- **답변 우수**: 심화 질문으로 엘리베이션 ("트래픽이 100배 늘어나면 어떻게 대응하시겠습니까?")
- **비언어 이벤트 연계**: 장기간 시선 회피 감지 시 "다른 방식으로 설명해보시겠어요?"

### Feature 2: 서버사이드 비언어/언어 분석 시스템

질문 세트별 녹화 영상을 S3에 업로드하면 Lambda가 자동으로 분석을 수행한다.

#### 분석 파이프라인

```
S3 PutObject (WebM) → EventBridge → Analysis Lambda 트리거
  1. FFmpeg: WAV 추출 + 프레임 추출
  2. OpenAI Whisper: 한국어 STT (타임스탬프 포함)
  3. GPT-4o Vision: 프레임 기반 비언어 분석 (시선, 표정, 자세)
  4. GPT-4o LLM: 언어 분석 (답변 구조, 논리성, 말투)
  5. 결과 통합 → Backend API로 피드백 저장
```

#### 분석 항목 및 피드백

| 분석 영역 | 분석 도구 | 분석 대상 | 피드백 예시 |
|----------|----------|----------|-----------|
| 시선 | GPT-4o Vision | 프레임 기반 시선 방향 | "시선이 불안정합니다" (eye_contact_score) |
| 표정 | GPT-4o Vision | 프레임 기반 감정 추정 | "표정이 경직되어 있습니다" (expression_label) |
| 자세 | GPT-4o Vision | 프레임 기반 상체 자세 | "자세가 기울어져 있습니다" (posture_score) |
| 언어 | GPT-4o LLM | Whisper STT 텍스트 | "답변 구조가 명확합니다" (verbal_score) |
| 불필요 단어 | GPT-4o LLM | STT 텍스트 패턴 | "음... 어... 반복이 많습니다" (filler_word_count) |

#### 기술 상세

- **Lambda Runtime**: Python 3.12, 2048MB 메모리, 최대 15분 타임아웃
- **FFmpeg Layer**: Static build 7.x (56MB Lambda Layer)
- **Whisper API**: `whisper-1` 모델, 한국어(`ko`) STT
- **GPT-4o Vision**: 추출된 프레임 이미지를 base64로 전달, 비언어 요소 분석
- **분석 진행 상태**: `STARTED → EXTRACTING → STT_PROCESSING → NONVERBAL_ANALYZING → VERBAL_ANALYZING → FINALIZING`

### Feature 3: 타임스탬프 기반 피드백 리뷰

> **리허설의 핵심 차별점.** 면접 녹화본을 재생하며 언어적/비언어적 피드백을 타임스탬프에 동기화하여 보여준다.

#### UI 구성

- **좌측 60%**: 비디오 플레이어 — 면접 녹화본 재생, 타임라인에 이벤트 마커 표시
- **우측 40%**: 피드백 패널 — 현재 재생 지점의 피드백 표시
- **하단**: 타임라인 내비게이션 — 언어적 이벤트(파란색 마커) + 비언어적 이벤트(주황색 마커) 구분 표시

#### 이벤트 마커 유형

| 색상 | 유형 | 예시 | 심각도 | 클릭 시 |
|------|------|------|--------|--------|
| 🔵 파란 | 언어적 피드백 | 답변 구조 부족, 핵심 누락 | 높음/중간/낮음 | 해당시점 이동 |
| 🟠 주황 | 비언어적 피드백 | 시선 이탈, 표정 경직, 자세 불안정 | 높음/중간/낮음 | 해당시점 이동 |
| 🟢 초록 | 우수 구간 | 명확한 설명, 좋은 자세 유지 | - | 해당시점 이동 |

#### 타임라인 동기화 로직

비디오 플레이어의 `currentTime`과 이벤트 배열의 `timestamp`를 매칭하여 재생 위치에 해당하는 피드백을 실시간으로 표시한다.

- 이벤트 마커 클릭 → 해당 시점으로 비디오 점프 + 우측 패널에 피드백 표시
- 비디오 재생 중 → 현재 시점의 피드백이 우측 패널에 자동 표시
- 양방향 동기화: 비디오 ↔ 타임라인 ↔ 피드백 패널

## 5. MVP 스코프 정의

### MVP DO (반드시 포함)

| # | 기능 | 세부 사항 |
|---|------|----------|
| 1 | AI 면접관 | 이력서 기반 + CS/시스템설계/BQ 질문 생성, 후속 질문 |
| 2 | 영상 녹화 + S3 업로드 | MediaRecorder API (WebM) → S3 Presigned URL 업로드 |
| 3 | 비언어 분석 | GPT-4o Vision (Lambda 서버사이드 프레임 분석) |
| 4 | STT 변환 | OpenAI Whisper API (Lambda 서버사이드) |
| 5 | 타임스탬프 피드백 | 비디오+피드백 동기화 UI, 이벤트 마커 타임라인 |
| 6 | 영상 변환 | MediaConvert (WebM→MP4, Lambda 트리거) |
| 7 | AI 피드백 생성 | GPT-4o LLM (Lambda) 답변+비언어 통합 분석 |
| 8 | 종합 리포트 | 면접별 요약 점수 + 개선포인트 + 강점 |

### MVP DON'T (후속 버전 연기)

- 기업별 질문 DB (RAG 기반)
- 사용자 면접이력 누적 대시보드
- 멀티유저 피어리뷰
- 모바일 앱 (웹 퍼스트)
- 결제 시스템 (베타 무료)
- 코딩 테스트 실행 환경 (IDE 통합)

### 전체 기술 스택

| 계층 | 기술 스택 | 선택 근거 |
|------|----------|----------|
| Frontend | React 18 + TypeScript 5 + Tailwind CSS + Vite | 최대 생태계, AI 코드 생성 품질, 빠른 HMR |
| 상태관리 | Zustand + TanStack Query | 경량화, 서버 상태 캐싱, 비디오 상태 관리 |
| 영상 녹화 | MediaRecorder API (WebM/VP9) | 브라우저 네이티브, 추가 라이브러리 불필요 |
| 실시간 STT | Web Speech API (ko-KR) | 면접 중 실시간 답변 인식 (후속질문용) |
| AI 질문 생성 | Claude API (claude-sonnet-4-20250514) | 질문생성 + 후속질문 |
| 분석 파이프라인 | OpenAI Whisper + GPT-4o (Lambda) | STT + 비언어 분석 + 언어 분석 통합 |
| 영상 변환 | AWS MediaConvert | WebM→MP4 (H.264+AAC, faststart) |
| 파일 저장 | S3 + Presigned URL | 영상 업로드/다운로드 |
| 이벤트 트리거 | EventBridge | S3 PutObject → Lambda 병렬 트리거 |
| Backend | Java 21 + Spring Boot 3.4 + Gradle (Kotlin DSL) | 한국 IT 표준, DI/AOP 성숙, 대규모 검증 |
| DB | MySQL 8.0 (prod) / H2 (dev) | CRUD 중심, Spring+JPA 최적 궁합 |
| ORM | Spring Data JPA + Hibernate | 타입 안전 쿼리, 자동 스키마 관리 |

### 데이터 플로우

```
[면접 진행 중]
MediaRecorder ──→ 질문세트별 영상 녹화 (WebM/VP9)

사용자 음성 ──→ Web Speech API STT ──→ 실시간 텍스트 (후속질문용)
                                          │
                                          ▼
                                    Claude API (streaming)
                                    ──→ 후속 질문 생성

[질문세트 완료 시]
FE: POST /upload-url ──→ BE: Presigned URL 발급
FE: PUT WebM → S3 (직접 업로드)
FE: POST /answers (질문별 startMs/endMs 타임스탬프)

[S3 업로드 감지 — 자동]
S3 PutObject ──→ EventBridge ──→ Lambda 병렬 트리거
  ├── Convert Lambda: WebM→MP4 (MediaConvert)
  └── Analysis Lambda:
        FFmpeg (WAV + 프레임 추출)
        → Whisper STT (ko)
        → GPT-4o Vision (비언어 분석)
        → GPT-4o LLM (언어 분석)
        → POST /feedback (타임스탬프별 피드백 저장)

[결과 조회]
FE: GET /status (폴링) → 분석 완료 확인
FE: GET /feedback → 타임스탬프 피드백 + Presigned URL (MP4 스트리밍)
```

### 핵심 데이터 구조

> 상세 스키마는 `docs/architecture/erd.md` 참조

주요 엔티티 관계:
- `interview` → `question_set` (1:N): 면접은 여러 질문 세트로 구성
- `question_set` → `question` (1:N): 질문 세트는 메인 질문 + 후속 질문 포함
- `question_set` → `file_metadata` (1:1): 질문 세트별 녹화 영상 파일
- `question_set` → `question_set_feedback` (1:1): 질문 세트별 AI 분석 피드백
- `question_set_feedback` → `timestamp_feedback` (1:N): 타임스탬프별 상세 피드백
- `timestamp_feedback`: 언어 점수(verbal_score) + 비언어 점수(eye_contact_score, posture_score, expression_label) + STT 텍스트(transcript) 통합

## 7. AI 프롬프트 아키텍처

> **RAG 없이 Claude의 기존 지식 + 잘 설계된 프롬프트로 MVP를 구현한다.**
> RAG는 기업별 질문DB나 사용자 이력 기반 피드백이 필요한 후속 버전부터 도입.

### 1단계: 질문 생성 프롬프트

- **입력**: 이력서 정보 + 면접 유형 + 난이도 설정
- **출력**: 면접 질문 5개 + 각 질문별 평가 루브릭
- **전략**: System prompt에 면접관 페르소나 설정 + 평가 기준 명시. Claude가 이미 알고 있는 CS 개념들을 활용하여 면접 질문 생성

### 2단계: 후속 질문 프롬프트

- **입력**: 이전 질문 + 사용자 답변 텍스트 + 평가 루브릭 + 비언어 이벤트(선택적)
- **출력**: 후속 질문 1개 + 이유
- **전략**: 답변 품질에 따라 분기 (심화/재질문/칭찬 후 다음 주제). 비언어 이벤트 연계하여 자연스러운 대화 유도

### 3단계: 통합 피드백 프롬프트

- **입력**: 전체 면접 대화 로그 + 전체 비언어 이벤트 배열 + 평가 루브릭
- **출력**: 타임스탬프별 피드백 배열 (JSON) + 종합 요약
- **전략**: 각 답변 구간에 대해 언어적 평가 + 해당 구간의 비언어 이벤트를 함께 분석하여 통합 피드백 생성
