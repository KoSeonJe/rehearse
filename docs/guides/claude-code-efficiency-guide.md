# Claude Code 토큰/컨텍스트 효율 사용 가이드

## 왜 필요한가?

Claude Code는 대화 기반 AI 도구로, **컨텍스트 윈도우**(한 번에 처리할 수 있는 텍스트 양)에 제한이 있다. 비효율적으로 사용하면 토큰이 빠르게 소진되고, 컨텍스트가 가득 차면 이전 대화 내용이 압축·손실된다. 이 가이드는 같은 작업을 **더 적은 토큰으로, 더 정확하게** 수행하는 방법을 정리한다.

---

## 1. 컨텍스트 윈도우 관리

| 명령 | 용도 | 언제 사용? |
|------|------|-----------|
| `/compact` | 대화 압축 (맥락 유지, 토큰 회수) | 컨텍스트 70% 이상 차면 |
| `/clear` | 대화 초기화 (완전 리셋) | 완전히 새 작업 시작할 때 |

### 원칙
- **한 세션 = 한 작업**에 집중. 관련 없는 작업은 새 세션(`/clear`)에서 시작
- 긴 디버깅/탐색 후 구현 시작 전에 `/compact`로 불필요한 탐색 로그 정리
- 작업 전환 시에도 `/compact` 고려

---

## 2. 프롬프트 작성법

### Bad — 토큰 낭비
```
코드 고쳐줘
```
- AI가 "어떤 코드?"를 파악하려고 프로젝트를 탐색 → 토큰 소모

### Good — 토큰 절약
```
frontend/src/hooks/use-interview.ts의 createInterview 함수에서
resumeText 파라미터를 필수에서 선택으로 변경해줘
```
- **파일 경로 명시**: 탐색 토큰 절약
- **함수명 명시**: 파일 전체를 읽지 않고 해당 부분만 처리
- **변경 내용 구체적**: 추가 질문 없이 바로 실행

### 추가 팁
- 여러 작업을 한 프롬프트에 넣지 않기 (컨텍스트 분산)
- 에러 메시지가 있으면 그대로 붙여넣기 (AI가 추측하지 않아도 됨)
- "전체 파일 보여줘" 대신 "X 함수만 보여줘" 사용

---

## 3. CLAUDE.md 활용

프로젝트 루트의 `CLAUDE.md`에 다음을 정리해두면 **매 세션마다 반복 설명이 불필요**:

- 기술 스택 (언어, 프레임워크, 버전)
- 코딩 컨벤션 (네이밍, 파일 구조, 금지 사항)
- 커밋/PR 규칙
- 프로젝트 구조 요약
- 자주 쓰는 명령어

`CLAUDE.md`는 **매 세션 시작 시 자동 로드**되므로, 한 번 잘 정리하면 모든 세션에서 효과를 본다.

---

## 4. 도구 사용 최적화

### 파일 탐색
| 상황 | 비효율 | 효율 |
|------|--------|------|
| 파일 찾기 | `find` 명령어 | `Glob` 패턴 (`**/*.tsx`) |
| 코드 검색 | `grep` 명령어 | `Grep` 도구 |
| 파일 읽기 | `cat` 전체 출력 | `Read` + offset/limit 또는 `Grep`으로 관련 부분만 |

### 에이전트 사용 기준
- **Glob/Grep으로 3회 이내 해결 가능** → 직접 도구 사용 (에이전트 불필요)
- **멀티스텝 탐색, 여러 파일 연쇄 분석** → 에이전트 사용
- 독립적인 검색은 **병렬 실행** 요청

## 5. /compact 타이밍 가이드

### 반드시 /compact 해야 할 때
1. 컨텍스트 70% 이상 경고가 나타났을 때
2. 긴 디버깅/탐색이 끝나고 구현을 시작하기 전
3. 하나의 작업이 완료되고 다음 작업으로 넘어갈 때

### /compact 전 체크리스트
- [ ] 현재 작업 상태를 MEMORY.md에 저장했는가?
- [ ] 다음 할 일을 기록했는가?
- [ ] 브랜치 정보를 기록했는가?

---

## 6. 메모리 시스템 활용

### MEMORY.md
- `~/.claude/projects/{프로젝트}/memory/MEMORY.md`에 위치
- 세션 간 맥락을 유지하는 핵심 도구
- `/compact` 전에 진행 상태 저장 → 압축 후에도 맥락 유지

### 저장할 것
- 현재 진행 상태, 완료된 작업
- 주요 아키텍처 결정 사항
- 디버깅에서 발견한 패턴/해결책
- 자주 사용하는 파일 경로

### 저장하지 말 것
- 임시 디버깅 로그
- 한 세션에서만 유효한 컨텍스트
- 추측이나 미검증 정보
