## 결정 목록

### ADR-001: 멀티 에이전트 워크플로우 혼합 실행 모드 도입
- **일자**: 2026-03-10
- **상태**: 승인
- **결정자**: 사용자 + Orchestrator

#### 문제
기능 개발 시 모든 단계를 자동으로 진행하면 중요한 설계 결정이 검토 없이 넘어갈 수 있고,
모든 단계를 수동 확인하면 개발 속도가 저하된다.

#### 선택지
1. 완전 자동 — 빠르지만 품질 리스크
2. 완전 수동 — 안전하지만 느림
3. 혼합 모드 — 핵심 단계만 수동, 나머지 자동

#### 결정
혼합 실행 모드 (선택지 3)

#### 근거
- Designer 결과물과 Backend API 설계는 이후 모든 작업의 기반 → 수동 확인 필수
- Frontend 구현, QA, DevOps는 이미 확정된 스펙 기반 → 자동 진행 가능
- Slack 알림으로 수동 확인 시점을 사용자에게 즉시 통보

#### 영향
- Orchestrator 워크플로우에 혼합 실행 플로우 추가
- Slack 알림 연동 필수
- 핸드오프 문서로 에이전트 간 컨텍스트 전달
