## 📊 주요 발견사항 (Key Findings)

### 1. 온보딩 플로우 패턴

#### 1.1 정보 수집 단계
| 서비스 | 수집 정보 | 단계수 | 특징 |
|--------|----------|--------|------|
| Yoodli | 이름, 사용 목적, 연습 모드 선택 | 3-4단계 | 시나리오 기반 선택 |
| Google Interview Warmup | 직무, 경력 수준 | 2단계 | 최소주의 (Minimal) |
| 사람인 AI | 직무, 경력, 자기소개서 (선택) | 3-4단계 | 이력서 기반 맞춤 |
| EnhanceU | 이름, 면접 코드 | 1단계 | 면접 관리자 중심 |

**리허설에 적용할 아이디어:**
1. **3단계 온보딩이 최적** — 너무 짧으면 맞춤도 낮고, 너무 길면 이탈률 증가
2. **이력서 선택** — 필수 아님, 선택사항으로 제공 (더 정확한 피드백을 원하는 사용자용)
3. **단계별 진행률 표시** — Progress Indicator(수직 또는 수평 스텝 표시)로 심리적 안정감 제공

#### 1.2 진행률 표시 (Progress Indicator) 패턴
- **스텝 인디케이터** (Step Progress): 원형 또는 숫자 버튼 → 단계가 명확한 온보딩에 최적
- **프로그레스 바** (Linear Bar): 작업 진행 중심 (예: 면접 진행 중)
- **원형 게이지**: 전체 진행률 시각화 (종합 리포트 점수 표시에 유용)

**리허설 적용:**
```
온보딩: [1 기본정보] → [2 카메라 테스트] → [3 시작 준비] (스텝 인디케이터)
면접 중: [Q1] → [Q2] → [Q3] (진행수 표시: "2/5 질문")
피드백: 원형 게이지 (전체 점수) + 카테고리별 수평 바 차트
```

#### 1.3 카메라/마이크 테스트 UI
**Yoodli 사례:**
- 카메라/마이크 선택 드롭다운 (명확한 라벨)
- 실시간 프리뷰 (사용자가 자신의 상태를 확인 가능)
- 볼륨 레벨 표시 (입력 장치 상태 가시화)
- "테스트 완료" 버튼으로 단계 진행

**리허설 개선점:**
1. **권한 요청 UI 최적화** — 브라우저 권한 창이 떴을 때 명확한 안내 텍스트
2. **에러 상태 명확히** — 카메라 미작동 시 구체적 해결 방법 제시
3. **재시도 가능성** — 테스트 중 실패해도 다시 할 수 있게 설계

### 2. 면접 진행 중 UI 패턴

#### 2.1 화면 레이아웃 구성
**표준 레이아웃 (3가지 주요 패턴):**

```
패턴 A: 카메라 중심 (대형)
┌─────────────────┐
│                 │
│   카메라 뷰     │  ← 70% 화면
│   (전체)        │
│                 │
└─────────────────┘
└─────────────────┘
│ 질문 텍스트     │
│ + 타이머        │  ← 30% 화면

패턴 B: 분할 화면 (추천)
┌──────────────┬──────────────┐
│              │ 질문 텍스트  │
│ 카메라 뷰    │ + 타이머     │
│ (50%)        │ + 다음 버튼  │ ← 50%
│              │ (50%)        │
└──────────────┴──────────────┘

패턴 C: AI 면접관 표시
┌──────────────┬──────────────┐
│ 카메라 뷰    │ AI 아바타    │
│              │ (애니메이션) │
│ (좌측)       │ + 질문 음성  │
│              │ (우측)       │
└──────────────┴──────────────┘
```

**리허설 현재 상태:** 패턴 A에 가까움
**권장 개선:** 패턴 B 또는 C로 전환 (사용자 집중력 향상)

#### 2.2 질문 표시 방식
| 방식 | 서비스 사례 | 장점 | 단점 |
|------|----------|------|------|
| **텍스트만** | interviewing.io | 읽기 속도 조절 가능 | 청각 장애인 제외 |
| **TTS (자동음성)** | Yoodli, Google Interview | 자연스러운 면접 | 목소리 톤 제한적 |
| **비디오 (AI 아바타)** | 고급형 서비스 | 실제 면접 시뮬레이션 | 개발 비용 높음 |
| **하이브리드** | 텍스트 + TTS + 버튼으로 재생 | 선택 가능 | UI 복잡도 증가 |

**리허설 적용:**
- 현재: TTS (Web Speech API) 사용 ✓
- 개선: TTS 재생 버튼 추가 (사용자가 몇 번이든 다시 들을 수 있게)

#### 2.3 타이머 및 시간 표시
**사용자 심리 고려:**
1. **시간 제한이 있을 때**: 초단위 카운트다운 (심리적 긴장감)
2. **자유로운 답변**: 경과 시간만 표시 (답변 길이 추적용)
3. **시각적 표시**:
   - 0-30초 남음: 빨강 배경
   - 30-60초: 주황색
   - 60초 이상: 회색

**리허설 현재:** 타이머 있음, 색상 변화 있음 ✓
**개선점:** 사운드 알림 추가 (마지막 10초 "핑핑" 음향)

#### 2.4 질문 간 전환 UX
**표준 패턴:**
- **자동 전환**: 사용자가 "완료" 또는 "다음" 버튼 누르면 자동 진행
- **애니메이션**: 질문 페이드 아웃 → 슬라이드 인 (부드러운 전환)
- **상태 유지**: 이전 답변 비디오는 재검토 가능하게 유지

**리허설 개선:**
1. **명확한 "다음 질문" 버튼** (배치: 화면 하단 우측, 코랄 액센트)
2. **애니메이션 효과** (0.3초 fade-in/out, 너무 화려하지 않게)
3. **진행 상황 표시** ("Q2/5" 형태로 명확히)

#### 2.5 후속 질문 (Follow-up) 표시
**Yoodli 사례:**
- AI가 사용자의 답변을 분석한 후 관련 후속 질문 생성
- 자동으로 다음 화면으로 전환되는 것이 아니라, 사용자가 "계속" 버튼을 눌러야 함
- 후속 질문임을 명확히 표시 ("이에 관해 조금 더 묻고 싶어요" 같은 톤의 텍스트)

**리허설 현재:** 후속 질문 구현 있음 ✓
**개선점:**
1. **톤 개선** — 딱딱한 기술적 표현 대신 따뜻한 톤으로
2. **시각적 구분** — 원래 질문과 후속 질문을 다른 배경색으로 표시

### 3. 피드백 및 분석 화면 패턴

#### 3.1 스코어/등급 표시
**서비스별 패턴:**

| 서비스 | 표시 방식 | 카테고리 수 | 특징 |
|--------|----------|----------|------|
| **하이잡** | 5등급(F~A) + 그래프 | 5가지 (구조, 명확성, 근거 등) | 명확하고 비교 용이 |
| **Yoodli** | 점수 + 원형 게이지 | 3-5가지 (Pace, Clarity, Confidence) | 목표 대비 현재 상태 표시 |
| **사람인** | 등급 + 진단 (인포그래픽) | 다양함 | 종합 리포트 형태 |
| **Google Interview** | 피드백 텍스트 중심 | 카테고리화 없음 | 실용적 조언 위주 |

**리허설 현재:**
- 5개 주요 지표 (구조, 명확성, 비언어, 음성, 자신감) 있음 ✓
- 점수 시스템 있음 ✓

**개선 아이디어:**
1. **혼합형 표시** — 수치(0-100) + 등급(Excellent/Good/Fair/Needs Work)
2. **진행도 시각화** — 이전 면접과의 비교 (트렌드 라인 그래프)
3. **카테고리별 색상 코딩** — 각 항목마다 시맨틱 컬러 사용
   ```
   구조:     파랑 (Info)
   명확성:   초록 (Success)
   비언어:   주황 (Warning) — 개선 필요 영역이 자주 나타나는 경향
   음성:     빨강 (Error) — 아니면 중립적으로
   자신감:   코랄 (Primary) — 리허설 액센트
   ```

#### 3.2 비언어 분석 시각화
**사용자 기대치가 높은 영역:**
- **표정 인식** (MediaPipe FaceLandmarker로 가능)
- **시선 처리** (노트북 카메라 기준 추정)
- **제스처/자세** (PoseLandmarker로 추적)
- **말하기 속도** (STT 타임스탐프 기반)
- **필러워드** (음성 인식 텍스트에서 탐지: "음", "어", "뭐", "저")

**표현 방식:**
1. **히트맵** — 얼굴 어디를 자주 볼 때 카메라를 응시했는지 시각화
2. **점수 바** — 각 항목별 0-100 스케일
3. **타임라인** — 영상 재생 시간축과 동기화된 피드백 포인트

#### 3.3 타임스탬프 동기화 피드백
**Google YouTube의 "챕터" 기능 참고:**
- 비디오 타임라인상에 마커 표시
- 마커 클릭 시 해당 시간으로 점프
- 마커 위에 호버하면 미리보기 (썸네일)

**리허설에서 구현할 구조:**
```
┌────────────────────────────────────┐
│  영상 플레이어 (16:9)              │
│                                    │
│  [Play] [Pause] [Mute] [FullScreen]│
└────────────────────────────────────┘
│ 타임라인 바 (0:00 ─────● ──── 3:42 │
│ 마커: ▼ @ 1:15 (구조 문제)         │
│       ▼ @ 2:03 (필러워드 감지)     │
│       ▼ @ 3:15 (긍정적 표현)       │
└────────────────────────────────────┘
┌────────────────────────────────────┐
│ 피드백 패널                        │
│ ┌──────────────────────────────┐  │
│ │ 1:15 - 구조 문제             │  │
│ │ "답변이 좋지만 도입부가      │  │
│ │  약간 불명확해요"           │  │
│ └──────────────────────────────┘  │
└────────────────────────────────────┘
```

#### 3.4 피드백 카드 레이아웃

### 4. 디자인 트렌드 분석

#### 4.1 컬러 스킴
**조사된 서비스들의 패턴:**
| 서비스 | 톤 | 주요색 | 특징 |
|--------|------|---------|------|
| Google Interview | 밝음 | Google Blue (#1F2937) | 최소주의 |
| Yoodli | 밝음 | 파랑/초록 혼합 | 활기찬 느낌 |
| Pramp | 중립 | 회색/검정 혼합 | 프로페셔널 |
| 사람인 | 밝음 | 코랄/주황 (한국 서비스) | 친근감 |
| EnhanceU | 밝음 | 파랑 (기업 느낌) | 신뢰성 강조 |

**리허설의 선택:**
- 밝은 배경 (Light Mode 기본) ✓
- 코랄 액센트 (#FF6B4A) — 토스, 한국 서비스 트렌드와 일치 ✓
- 다크모드 지원 계획 시 slate-900 배경 권장

**개선점:**
1. **일관된 색상 사용** — 같은 의미는 같은 색상으로 (신뢰성)
2. **충분한 명도 대비** — WCAG AAA 기준 충족 (접근성)
3. **다크모드** — 밤 시간 사용자 고려

#### 4.2 일러스트레이션/캐릭터
**조사 결과:**
- Google Interview Warmup: 일러스트레이션 거의 없음 (텍스트 위주)
- Yoodli: 심플한 아이콘 사용
- 한국 서비스: 캐릭터/마스코트 자주 등장
- Pramp: 추상적 도형/그래프 중심

**리허설 현재:**
- 거울 앞의 캐릭터 로고 설계됨 ✓
- 다양한 표정 바리에이션 계획 ✓

**개선:**
1. **로딩 상태**: 캐릭터가 생각하는 애니메이션
2. **성공 상태**: 캐릭터가 웃는 애니메이션 (Lottie JSON 애니메이션 추천)
3. **에러 상태**: 캐릭터가 고민하는 표정
4. **온보딩**: 각 단계별로 캐릭터의 톤 변화 (격려, 준비, 도움)

#### 4.3 마이크로 인터랙션
**표준 패턴:**
1. **호버 효과** — 버튼 배경색 변화, 약간의 스케일 업 (scale 1.02)
2. **활성 상태** — 누른 버튼의 깊이감 (box-shadow 또는 배경 어두워지기)
3. **로딩 상태** — 스피너 또는 퍼지 효과
4. **성공 피드백** — 체크마크 애니메이션 + 짧은 사운드
5. **스크롤 애니메이션** — 요소가 나타날 때 fade-in/slide-up

**리허설 추천:**
```javascript
// 버튼 호버
.btn:hover {
  transform: scale(1.02);
  box-shadow: 0 2px 8px rgba(255, 107, 74, 0.2);
  transition: all 200ms ease-out;
}

// 진행도 바 채우기 애니메이션
@keyframes fillBar {
  from { width: 0; }
  to { width: var(--progress); }
}
```

## 🎯 리허설에 적용 가능한 구체적 개선안

### Phase 1: 온보딩 개선 (우선순위: HIGH)

#### 1. 스텝 프로그레스 UI 추가
```
목표: 사용자가 현재 어디쯤인지 명확히 파악
구현:
- 3-4단계 스텝 인디케이터 (원형 또는 숫자)
- 현재 단계: 코랄 채움, 이전 단계: 초록 체크마크, 다음 단계: 회색
- 각 단계명: "기본정보" → "카메라테스트" → "시작준비"
- 스텝 사이 연결선 (현재 진행도를 선의 굵기로 표현 가능)

디자인 토큰:
- 활성(코랄): #FF6B4A
- 완료(초록): #00C48C
- 비활성(회색): #AEB5BC
```

#### 2. 카메라/마이크 테스트 화면 개선
```
현재 문제: 권한 요청 후 바로 녹화 시작이라 사용자가 불안감 느낌

개선안:
1. "카메라 테스트" 명시적 단계 추가
2. 실시간 프리뷰 제공 (카메라 피드 표시)
3. 마이크 음량 레벨 시각화 (0-100 바 또는 네 개의 점)
4. "테스트 완료" 버튼으로 진행
5. 에러 메시지:
   - "카메라가 감지되지 않습니다" → "해결 방법: 브라우저 설정 → 카메라 권한 확인"
   - "마이크가 작동하지 않습니다" → "다른 앱이 마이크를 사용 중인지 확인하세요"
```

#### 3. 톤 개선 (따뜻한 말투)
```
변경 전: "카메라 권한을 허용해주세요"
변경 후: "면접관이 당신을 볼 수 있도록 카메라를 켜주세요"

변경 전: "테스트 시작"
변경 후: "함께 확인해봅시다"
```

### Phase 2: 면접 진행 중 UI 개선 (우선순위: HIGH)

#### 1. 화면 레이아웃 변경 (패턴 B: 분할 화면)
```
현재 (패턴 A):
┌──────────────────┐
│   카메라 뷰      │ 100%
│  (전체 화면)     │
└──────────────────┘
└──────────────────┘
│ 질문 + 타이머    │

개선안 (패턴 B - 권장):
┌──────────────┬──────────────┐
│              │ AI 질문      │
│ 카메라 뷰    │ "다음으로    │
│ (50%)        │ 프로젝트에   │
│              │ 대해 설명해  │
│ [녹화중]     │ 주세요"      │
│              │              │
│              │ ⏱ 0:45       │
│              │              │
│              │ [완료] [건너뛰기]
└──────────────┴──────────────┘

장점:
- 사용자가 자신의 얼굴 + 질문을 동시에 볼 수 있음
- 타이머가 눈에 잘 들어옴
- 버튼 배치가 명확
- 데스크톱/태블릿에서 최적
```

#### 2. 질문 표시 강화
```
변경 전:
"Tell me about your background"

변경 후 (멀티 모달):
┌─────────────────────────────┐
│ AI가 말하는 중...          │
│ [🔊 다시 듣기] [🔕 음소거] │
└─────────────────────────────┘

텍스트:
"지금까지의 경력을 소개해 주시겠어요?"

특징:
- TTS 동시 재생
- 재생 버튼으로 몇 번이든 듣기 가능
- 음소거 옵션 (음성만 듣고 싶은 사용자)
```

#### 3. 진행 표시 명확화
```
변경 전:

변경 후 (더 눈에 띄게):
┌──────────────────────────────┐
│ 2 / 5 질문 완료              │
│ ███████░░░░░░░░░░░░░░░░░ 40%│
└──────────────────────────────┘

또는 다음과 같이 시각적으로:
[●] 1 [●] 2 [●] 3 [◯] 4 [◯] 5
     ✓   ✓   ▶

색상:
- 완료: 초록
- 현재: 코랄
- 남은 것: 회색
```

#### 4. 후속 질문 톤 개선
```
변경 전:
"Follow-up question: ..."

변경 후 (따뜻한 톤):
"이 점에 대해 좀 더 자세히 들어봐도 될까요?"

시각적 구분:
- 배경색: Info Light (#EBF4FF) 또는 아주 옅은 파랑
```

```
│ 💡 피드백:                      │
```

#### 2. 비언어 분석 시각화
```

구현:
1. 타임라인 동기화 피드백
   - 영상 재생 시 특정 시간의 피드백이 화면에 팝업
   - 예: "1:23 - 눈 마주침이 좋습니다" 라는 바뀐 카드

2. 히트맵 시각화 (선택)
   - 얼굴의 어느 부분을 자주 응시했는지 색상으로 표시
   - 카메라를 자주 바라본 시간대를 초록색으로 표시
   - 불안정한 시간대는 주황색

3. 자세 분석
   - MediaPipe PoseLandmarker로 어깨 높이, 앉은 자세 추적
   - "좋은 자세: 어깨가 긴장하지 않았습니다" 같은 긍정적 피드백

4. 필러워드 카운트
   - "음", "어", "뭐", "저" 등 탐지
   - 그래프: 질문별로 필러워드 수 비교
   - 트렌드: 면접 진행할수록 줄어드는 것을 보여줌
```

### Phase 4: 디자인 시스템 강화 (우선순위: MEDIUM)

#### 1. 컴포넌트 추가 (현재 `.omc/plans/2026-03-10-ui-redesign.md`에 이미 있음)
```
새로 추가할 컴포넌트:

1. ProgressIndicator (스텝)
   - Variants: 3단계, 4단계, 5단계
   - States: inactive, active, completed
   - 마크업: <StepProgress current={2} total={4} />

2. ScoreCard (점수 카드)
   - Props: title, score, maxScore, trend, feedback
   - 배경색 시맨틱 기반

3. FeedbackBubble (타임라인 피드백)
   - Props: timestamp, message, type (positive/neutral/improvement)
   - 아이콘 + 텍스트 + 타임 링크

4. AnalysisChart (분석 차트)
   - Bar chart (항목별 점수)
   - Line chart (시간대별 변화)
   - Gauge (원형 게이지 — 종합 점수)
```

#### 2. 모션/애니메이션 가이드
```
지속시간 표준:
- 빠른 피드백: 150ms (호버, 버튼 클릭)
- 보통 전환: 300ms (페이드, 슬라이드)
- 느린 진입: 500ms (모달, 로딩 시작)

이징 함수:
- ease-out: 버튼 클릭, 완료 상태
- ease-in-out: 일반적 전환
- custom cubic-bezier(0.4, 0, 0.2, 1): 머티리얼 디자인 표준

금지사항:
- 과도한 애니메이션 (3초 이상)
- 자동 반복 (사용자 주의 방해)
- 깜빡임 (epilepsy 위험)
```
