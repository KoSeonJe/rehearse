# 랜딩 카피 시트 — 2026-04-14

## 개요

Rehearse 랜딩 페이지 전면 리뉴얼 카피. 8개 섹션 × Before/After + 근거. 타임스탬프 피드백(영상 ↔ 마커 ↔ 패널 3-way sync) 중심의 차별화 강조.

---

## Section 1: Hero
**목적**: 3초 내에 제품 핵심 전달. "영상 복기 도구"임을 명확히.

**Before**:
```
면접, 연습하면 달라집니다.

이력서 분석은 물론, CS 기초와 직무 지식까지
AI가 맞춤 질문을 만들고 영상과 함께 피드백합니다.

[무료로 시작하기]
GitHub 또는 Google 계정으로 30초 만에 시작
```

**After**:
- **헤드라인**: "면접 영상을 마커로 클릭하며 복기하는 도구"
- **서브카피**:
  ```
  영상의 정확한 순간마다 피드백을 확인하세요.
  0:23에 어깨가 말렸다, 1:42에 목소리 톤이 올라갔다.
  ```
- **CTA 버튼**: "지금 시작하기" (또는 "첫 면접 연습 해보기")
- **보조 문구**: "무료, 30초 가입, Chrome 브라우저만 필요"

**근거**:
- "AI가 맞춤" → 금지어 제거. 사용자 주어로 전환.
- 서브카피는 차별점(타임스탐프 + 구체적 피드백) 명시. 경쟁사(Final Round AI, Huru)는 점수표만 제공하므로 직접 대비 가능.
- 헤드라인 ≤ 20자 충족 (18자).

## Section 2: Pain Points (새 섹션)
**목적**: 취준생의 면접 피드백 불만 3가지. 문제 인식 → 솔루션 연결.

**구성**:
```
타이틀: 면접 끝나면 뭘 못했는지 모르겠다

[3개 카드 좌우 배치]

1. "점수와 논평만 받는다"
   ├─ 문제: Final Round AI, Huru 같은 도구는 전체 점수만 표시
   ├─ 대사: "종합 점수 7.2점이라는데, 어디를 개선해야 하는지 몰라요."
   └─ 해결: Rehearse는 0:42에 이렇게 좋았다, 1:15에 이렇게 개선하세요 명시

2. "영상과 피드백이 동떨어졌다"
   ├─ 문제: 영상을 보면서 피드백을 읽기 어려움
   ├─ 대사: "영상을 봐도 어느 부분의 피드백인지 헷갈려요."
   └─ 해결: 마커를 클릭하면 그 순간 영상으로 이동 (3-way sync)

3. "비언어 분석은 있지만 구체적이지 않다"
   ├─ 문제: "자신감 부족" 같은 추상적 피드백만 받음
   ├─ 대사: "표정이 어떻게 부족한지 구체적으로 모르겠어요."
   └─ 해결: 프레임 단위 9지표 분석 (postureLevel, expressionLabel, eyeContactLevel, speechPace, toneConfidenceLevel, emotionLabel, fillerWordCount, verbal_score, accuracy_issues) 제시
```

**카피 톤**: 사용자의 목소리 인용. "~어요", "~모르겠어요" (공감톤).

## Section 3: Timestamp Demo (기존 'Key Features' 재구성)
**목적**: "이 마커를 클릭하면 그 순간으로" 인터랙티브 경험. 피드백 샘플 3개(언어/비언어/우수).

**구성**:
```
타이틀: 영상 한 장면, 세 가지 피드백

[좌측: 영상 + 타임라인 마커 3개]
[우측: 마커별 피드백 카드]

마커 1 (0:42, 언어): 초록 마커
├─ 타입: 잘한 점
├─ 피드백 샘플:
│  "기술적 부채 해결 경험을 구체적으로 잘 설명하셨어요. 
│   STAR 기법(상황→과제→행동→결과)이 자연스럽게 녹아있습니다."
└─ 사용자 언어: 구체적 칭찬 + 개선 불필요

마커 2 (1:15, 비언어): 주황 마커
├─ 타입: 개선 포인트
├─ 피드백 샘플:
│  "0:42~1:15 구간에서 시선이 화면 아래로 자주 향했습니다. 
│   카메라를 바라보며 답변하면 자신감이 더 전달돼요."
└─ 사용자 언어: 시간 범위 + 구체적 개선책

마커 3 (2:30, 우수): 파랑 마커
├─ 타입: 기술적 정확도
├─ 피드백 샘플:
│  "HashMap과 ConcurrentHashMap의 차이 설명이 정확합니다. 
│   멀티스레드 환경에서의 성능 차이까지 언급한 점이 돋보여요."
└─ 사용자 언어: 기술 정확도 + 심화 이해 인정
```

**시각 설명**:
- 영상 재생 중 마커 3개 노출 (색상 코드: 파랑=언어, 주황=비언어, 초록=우수)
- 마커 클릭 → 영상 해당 시점으로 스크롤 + 피드백 카드 활성화
- 3-way sync 강조: "영상 ↔ 마커 ↔ 피드백 패널이 함께 움직여요"

## Section 4: Metrics (9 지표 × 타임스탐프)
**목적**: 비언어 + 언어 지표 8개를 사용자 언어로 설명. "복잡한 AI 분석"이 아닌 "사용자가 이해할 수 있는 피드백".

**구성**:
```
타이틀: 답변의 매 순간을 분석하는 9가지 지표

[2단 구성: 언어 4개 + 비언어 5개]

## 언어 피드백 (파랑)

1. **verbal_score** (답변 완성도)
   한 줄: "질문에 얼마나 정확히 답했는가?"
   예시: "HashMap 설명이 정확하고 충분했습니다."

2. **accuracy_issues** (기술적 오류)
   한 줄: "기술 용어나 개념에 오류가 있었는가?"
   예시: "Thread-safe한 컬렉션 설명에 오류가 없어요."

3. **fillerWordCount** (반복사, 음성 습관)
   한 줄: "어, 음, 그, 뭐 같은 반복사가 몇 개?"
   예시: "전체 답변 중 반복사가 2번 나왔어요. 자연스럽습니다."

4. **speechPace** (말하기 속도)
   한 줄: "빨랐나, 느렸나, 적당했나?"
   예시: "평균 초 당 단어 수가 3.2개. 충분히 신중하게 설명했어요."

## 비언어 피드백 (주황 + 초록)

5. **postureLevel** (자세 안정성)
   한 줄: "몸이 흔들렸거나 웅크렸나?"
   예시: "처음부터 끝까지 안정적인 자세를 유지했습니다. 자신감 있어 보여요."

6. **expressionLabel** (표정 변화)
   한 줄: "표정이 딱딱했나, 자연스러웠나?"
   예시: "미소가 자연스러워서 친화력이 느껴집니다."

7. **eyeContactLevel** (시선 처리)
   한 줄: "카메라를 얼마나 바라봤나?"
   예시: "시선 방향이 화면 아래쪽이 많았어요. 카메라 정면을 더 자주 보세요."

8. **toneConfidenceLevel** (음성 톤의 자신감)
   한 줄: "목소리가 떨렸나, 확신감 있나?"
   예시: "음성 톤이 안정적이고 발성이 명확해요. 자신감이 전달됩니다."

9. **emotionLabel** (감정 표현)
   한 줄: "긍정적/중립적/불안했나?"
   예시: "전반적으로 침착하고 긍정적인 태도가 느껴집니다."
```

**카피 톤**: 모두 "한 줄 사용자 질문" + "예시(칭찬 또는 개선책)". 학술적 용어 최소화.

## Section 5: Dev Tailored (이력서 → 질문 → 후속 3라운드)
**목적**: "개발자 면접에만 특화"임을 구체적으로. CS/시스템설계/직무 맞춤.

**타이틀**: "개발자 이력서로 CS·직무·설계 질문을 생성합니다"

**구성**:

```
## 라운드 1: 이력서 분석 기반 질문
상황: 지원자가 "Spring Boot + JPA + Redis 경험" 이력서 업로드
생성 질문 (3개 예시):
- "HashMap과 ConcurrentHashMap의 차이를 설명하고, 
   각각 언제 사용하는지 말씀해주세요." [CS 기초]
- "Spring Boot에서 트랜잭션 전파 전략을 어떻게 설계하셨나요?" 
  [직무 지식]
- "팀에서 코드 리뷰 프로세스를 개선한 경험이 있다면 
   말씀해주세요." [이력서 기반]

## 라운드 2: 심화 질문 (최대 3라운드)
지원자의 첫 답변을 Claude가 분석 후:
- **심화**: "그럼 Redis 캐시 전략에서 TTL을 어떻게 설정하셨나요?"
- **명확화**: "ConcurrentHashMap의 segment 기반 락이라고 하셨는데, 
   그 원리를 다시 한번?"
- **반론**: "멀티스레드 환경에서 성능이 중요하다면, 
   왜 ConcurrentHashMap이 HashMap보다 느릴 수 있나요?"
- **적용**: "당신이 설계한 캐시 전략이라면, 
   메모리 부족 상황을 어떻게 대처하시겠어요?"

## 라운드 3: 문제 해결 (선택 사항)
경우에 따라 시스템설계 미니 문제 제시:
- "일일 1억 건의 API 요청을 처리하는 아키텍처를 설계하세요"
- "Redis 캐시 무효화 전략의 trade-off는 뭐라고 생각해요?"
```

**카피 톤**: "AI가 당신의 이력서를 읽고" → 구체적 예시 제시. 실제 생성 질문 노출.

## Section 6: Journey (4단계)
**목적**: Setup → Preparation → Interview → Feedback 순서. 각 단계 ≤ 20자 설명.

**타이틀**: "당신의 면접 여정, 4단계"

**구성**:

| 단계 | 제목 | 설명 (≤20자) | 상세 |
|------|------|-------------|------|
| 1 | 나만을 위한 면접 설정 | 직무·경력·이력서 선택 | AI가 맞춤 질문 세트 준비 |
| 2 | 몰입도 높은 AI 면접 | AI와 실전처럼 대화 | 음성 감지 → 자동 다음 질문 |
| 3 | 매 순간을 분석하는<br/>타임스탐프 피드백 | 영상+마커+패널 동시 | 언어, 비언어, 기술 정확도 분석 |
| 4 | 개선점을 찾고 다시 도전 | 피드백 정리 후 재도전 | (명시적으로 언급하지 말고 암시) |

**카피 톤**: 각 단계 헤드라인은 사용자 입장 (능동태). "당신이 설정한다" "당신이 면접본다" "당신이 확인한다".

## Section 7: FAQ (오해 정리형 6개)
**목적**: 금지 오해 6가지를 명확히. "이게 아니다"를 강하게.

**타이틀**: "흔한 오해, 명확히 해드릴게요"

**Q&A**:

| # | 질문 | 답변 |
|---|------|------|
| 1 | **실시간 코칭인가요?** | 아니요, 사후 분석입니다. 면접 후에 영상을 보며 피드백을 확인합니다. 실시간 조언은 없어요. |
| 2 | **종합 점수를 주나요?** | 점수를 드리지 않습니다. 대신 "0:42에 이렇게 좋았고, 1:15에는 이렇게 개선하세요"라는 구체적 피드백을 제공합니다. |
| 3 | **시선 추적(Eye tracking) 기술인가요?** | 아니요, 시선 위치(up/down/left/right)만 감지합니다. 안경, 조명, 카메라 각도에 따라 정확도가 달라질 수 있습니다. |
| 4 | **이력서는 어디에 저장되나요?** | 질문 생성에만 사용된 후 즉시 삭제됩니다. 별도로 보관하지 않습니다. |
| 5 | **녹화된 영상은 얼마나 오래 보관되나요?** | 안전한 클라우드 스토리지(AWS S3)에 저장되며, 본인이 언제든 삭제할 수 있습니다. |
| 6 | **모바일에서도 사용할 수 있나요?** | 데스크톱 Chrome 브라우저에 최적화되어 있습니다. 태블릿도 가능하지만, 마이크·카메라 권한을 허용해야 합니다. |

**카피 톤**: "아니요"로 명확히 시작. "대신 ~입니다"로 긍정적 기능 강조.

## Section 8: Final CTA
**목적**: 한 문장 + 버튼. 마지막 확신 주기.

**타이틀**: (없음, 단순 헤드라인)

**카피**:
```
준비된 만큼 보여줄 수 있습니다.

지금 바로 리허설을 시작하고,
당신의 면접 영상을 한 프레임씩 복기해보세요.

[지금 시작하기]
```

**버튼 대안 3가지**:
1. "지금 시작하기" (기본, 가장 범용적)
2. "첫 면접 영상 녹화하기" (구체적 행동 유도)
3. "무료로 시작해보기" (비용 부담 없음 강조)

**카피 톤**: "준비된 만큼" = 사용자의 성과, 우리는 도구만 제공. 겸손하면서 강력.
